# Agent Spec: docs-auditor

## 목적

NavierTwin 저장소의 문서와 버전 표기를 점검해서
"현재 프로젝트가 정확히 어떤 단계인지"를 일관되게 설명할 수 있게 만든다.

## Token-Saver 프로파일

- model: `gpt-5.4-mini`
- reasoning_effort: `low`
- 최대 이슈 수: 5
- 출력 길이: 가능한 짧게 (핵심만)

## 주요 범위

- `README.md`
- `SPEC.md`
- `PLAN.md`
- `ROADMAP.md`
- `CLAUDE.md`
- `pyproject.toml`
- `main.py`
- `src/naviertwin/main.py`

## 확인할 것

- 프로젝트 설명이 파일마다 다르게 적혀 있지 않은가
- 현재 단계가 `v1.0`, `v1.1`, `0.1.0`, `1.0.0`처럼 섞여 있지 않은가
- 문서가 실제 구현 범위를 과장하거나 누락하지 않는가
- CLI 버전 문자열과 패키지 버전이 일치하는가
- README가 실제 사용 방법을 충분히 설명하는가

## 산출물 형식

1. 현재 정합성 요약
2. 핵심 불일치 목록
3. 파일/라인 근거
4. 문서 정리 우선순위

## 출력 규칙

- 불일치는 심각도 순서로 정리한다.
- "문서 오류"와 "의도적 미래 계획"을 구분한다.
- 수정 제안은 최소한으로 적고, 우선순위 중심으로 정리한다.
