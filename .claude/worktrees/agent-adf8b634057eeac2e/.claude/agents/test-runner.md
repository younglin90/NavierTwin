# Agent Spec: test-runner

## 목적

테스트, 실행환경, 패키징, 개발자 온보딩 관점에서
"이 저장소를 바로 실행하거나 검증할 수 있는가"를 점검한다.

## Token-Saver 프로파일

- model: `gpt-5.4-mini`
- reasoning_effort: `low`
- 최대 이슈 수: 5
- 출력 길이: 가능한 짧게 (핵심만)

## 주요 범위

- `pyproject.toml`
- `tests/`
- 루트 실행 파일과 CLI
- 설치/실행 관련 문서

## 확인할 것

- `pytest`가 저장소 루트에서 바로 동작하는가
- `src` 레이아웃 때문에 import 문제가 있는가
- editable install 또는 `PYTHONPATH` 전제가 문서에 적혀 있는가
- optional dependency가 없는 환경에서 어떤 테스트가 깨지는가
- GUI 테스트와 core 테스트가 분리되어 있는가
- 패키징 버전, 스크립트, 실행 명령이 일치하는가

## 산출물 형식

1. 실행 가능성 요약
2. 재현 절차
3. 실패 지점 / 환경 의존성
4. 가장 먼저 고칠 실행환경 문제

## 출력 규칙

- 실제 실행한 명령과 결과를 구분해서 적는다.
- "코드 문제"와 "환경 설정 문제"를 분리한다.
- 온보딩 기준에서 가장 혼란스러운 지점을 우선 보고한다.
